Breaking Changes

Inventory DMDEV-89794

The exceedingly old and replace pair of tables, tbl_ArchiveInventory and tbl_ArchiveInventoryStatus, having been deprecated since JE 3.5.2.2, are explicitly excluded from any Purge activities. In addition, the database Foreign Key linking the former to the core tbl_Inventory table has been removed (DROPped) and will not be reinstated. Any Custom Code still referencing this tables ought to be eliminated entirely.

Any home-brew, or JE Support-developed Inventory Purge procedures must be removed immediately and never reinstated. The new Inventory Purge supports purge of data from both TG JE core and any custom tables using InventoryID. Using any other Purge mechanism is unsupported and is undertaken at the user’s own risk.
