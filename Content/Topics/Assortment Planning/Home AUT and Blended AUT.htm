Home AUT and Blended AUT

AP distinguishes between two different types of AUT (or Average Unit Ticket, for example, the ticket price intended to be printed on merchandise tags). The first of these is Home AUT. This is meant to represent the AUT value for the country and currency of the defined Home Site (set in the Manage Assortment Plans screen for AP). This is the starting point for planning AUT, as this value is user-editable. Home AUT is only editable - and only has a value - in future periods.

From this, Blended AUT is derived. Blended AUT is considered to be a true representation of the expected earnings for a given item. Blended AUT incorporates several components, each of which is enabled by system policy:

  • VAT Rate changes between the Home Site and sites within each cluster

  • Price Exception Factors

  • Price Matrix conversions

Only Price Exceptions OR Price Matrix conversion can be enabled at once (but both can be disabled altogether if desired). VAT Rate changes cannot be enabled if Price Matrix conversions are enabled.

Blended AUT is derived in future periods from Home AUT. In past periods, Home AUT is empty, and Blended AUT is derived from provided sales history.

Metric Definition
Home AUT AUT as printed on the ticket in the home currency, it is VAT inclusive (but the VAT rate of the Home Site specifically)
Blended AUT AUTs of all currencies, converted back to home currency, and weighted-averaged together

Price Matrix functionality

If a user enables Price Matrix conversions via system policy Price Conversion Method, and sets a Price Matrix on a given placeholder, then the configured Price Matrix is used to convert the provided Home AUT value to Blended AUT.
